# Branchen-Beispiele für `audit-rules.md`

Diese Files sind **Inspiration für den Skill-Nutzer (Händler oder Agentur), nicht Vorgabe für den Audit-Lauf.**

## Wann der Skill diese Files lädt

- Wenn der User explizit nach einem Beispiel fragt: „zeig mir wie sowas für Fashion aussieht", „gib mir ein Beispiel für Skincare-Stores".
- Wenn der User im Audit-Rules-Interview (siehe `../audit-rules-interview.md`) eine Inspiration für eine bestimmte Frage braucht — dann **die zur Branche passende Datei zitieren**, immer mit Hinweis „so machen das andere Stores, deine Werte können anders sein".

## Wann der Skill diese Files NICHT lädt

- **Nie** 1:1 als `audit-rules.md` ins Projekt kopieren.
- **Nie** stillschweigend als Default verwenden, wenn der User keine eigenen Regeln angibt.
- **Nie** Werte aus diesen Files (z.B. „min. 3 Bilder", „60 Zeichen Meta-Title") als bestätigte Audit-Schwellen behandeln.

Der Grund: Jeder Store hat andere Lieferanten, andere Kanäle (B2C / B2B / Marketplace), andere SEO-Strategien. Eine Fashion-Regel wie „Title-Pattern `{Brand} {Name} - {Kategorie}`" ist für einen Single-Brand-Store falsch.

## Inhalt

- `example-fashion.md` — DTC-Apparel-Store, mehrere Brands, klassisches B2C
- `example-skincare.md` — Cosmetics / Personal Care, INCI-Felder, Pflege-Specs
- `example-b2b-tools.md` — Industriebedarf, Hersteller + Artikelnummer im Titel

Neue Beispiele aufnehmen, wenn ein wiederkehrendes Branchen-Pattern auftaucht.
